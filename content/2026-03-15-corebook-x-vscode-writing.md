## なぜ執筆にVS Codeなのか？：エンジニア流の文章術

世の中には、WordやGoogleドキュメント、あるいはNotionやObsidianといった優れた執筆ツールが溢れています。しかし、私はあえて**VS Code**を選択しています。理由はシンプルです。

1. **Markdownとの親和性:** このブログ（Pelican）のソースはMarkdownです。VS Codeは世界で最も優れたMarkdownエディタの一つです。
2. **Gitによるバージョン管理:** 記事を「コード」として管理し、GitHubにプッシュすることでバックアップと履歴管理を完結させられます。
3. **拡張性:** 必要に応じて、自分好みの執筆補助ツールを無限に追加できます。
4. **CoreBook Xのパワーの活用:** 多数の拡張機能を動かしても、Ryzen 5なら一切の遅延なく動作します。
5. **3:2ディスプレイの最大活用:** 縦に長い画面は、テキストエディタを表示するのにこれ以上ないほど適しています。

## 執筆を支える「三種の神器」拡張機能とカスタマイズ

私がVS Codeを「執筆モード」で使う際、欠かせない拡張機能と設定について深掘りします。

### 1. Markdown All in One
これなしでのMarkdown執筆は考えられません。
- **キーボードショートカット:** `Ctrl + B` で太字、`Ctrl + I` で斜体など、プログラミング時の操作感をそのまま文章に適用できます。
- **自動リスト補完:** 行頭で `1.` や `-` を入力してエンターを押すと、次の行も自動的にリスト形式になります。
- **目次の自動生成:** 長文になればなるほど、全体像の把握が難しくなりますが、これなら一瞬で `Table of Contents` を作成・更新でき、読者の利便性も向上します。

### 2. Prettier - Code formatter
「文章にフォーマッタ？」と思われるかもしれませんが、これこそが爆速の鍵です。
改行のルール、リストのインデント、Markdownの表（Table）の崩れなどを、保存時（`Ctrl + S`）に自動で美しく整えてくれます。書き手は「見た目」を気にすることなく、ただ「思考の吐き出し」に集中できるのです。CoreBook XのRyzen 5は、巨大なMarkdownファイルの整形もコンマ数秒で終わらせ、思考のテンポを崩しません。

### 3. zen-mode (禅モード) とエディタ設定
`Ctrl + K Z` で起動する禅モードは、CoreBook Xの3:2ディスプレイと最高の相性を見せます。余計なUIを消し去り、画面の中央にエディタだけを表示する。さらに私は以下の設定を `settings.json` に加えています。

```json
{
  "editor.fontSize": 18,
  "editor.lineHeight": 2.0,
  "editor.wordWrap": "on",
  "editor.minimap.enabled": false,
  "editor.cursorSmoothCaretAnimation": "on",
  "editor.smoothScrolling": true
}
```
この「ゆったりとした行間」と「滑らかなカーソル移動」が、CoreBook Xの高精細な画面と相まって、まるで高級な万年筆で紙に文字を書いているかのような錯覚を覚えさせます。

## AIとの共作：GitHub Copilotは「もう一人の自分」

2026年の今、執筆においてAIを無視することはできません。私は **GitHub Copilot** を執筆のパートナーとして迎え入れています。

Copilotはコードだけでなく、日本語の文章の補完も驚くほど優秀です。「なぜなら、」と入力した瞬間に、これまでの文脈を読み取って論理的な続きを提案してくれる。あるいは、技術的な用語の説明を求めれば、正確な定義を提示してくれる。

もちろん、AIが書いた文章をそのまま使うことはありません。しかし、**「白紙を前にして固まる時間」**を、Copilotが最小限にしてくれるメリットは計り知れません。Copilotが提示した「叩き台」を、自分の言葉で洗練させていく。このプロセスは、一人で暗闇を歩くような執筆作業を、信頼できる相棒との楽しい対話に変えてくれました。

Ryzen 5 7430Uの処理能力は、VS Code上でCopilotの提案をリアルタイムに受け取る際も、一切のストレスを感じさせません。バックグラウンドで走るAIの推論と、私のタイピングが完璧に同期しているのです。

## 自動化の極致：GitHub ActionsによるCI/CD

記事を書き終えた後の「公開作業」も、CoreBook Xならエレガントに完結します。

私はGitHubのリポジトリに `.github/workflows/deploy.yml` を配置し、`main` ブランチにプッシュした瞬間に自動でブログが更新される仕組みを作っています。
- **Pelicanビルド:** GitHub Actions上のUbuntuコンテナでPythonが走り、MarkdownをHTML化。
- **自動デプロイ:** 生成されたHTMLが自動的にサーバー（GitHub PagesやVercelなど）に反映されます。

CoreBook X側では、`git push` を叩くだけ。あとはAIと自動化の仕組みが、私の言葉を世界中へ届けてくれます。この「システムを構築する楽しさ」も、エンジニアがブログを書く醍醐味の一つです。
